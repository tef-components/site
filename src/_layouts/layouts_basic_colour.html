Brand colors allow the user to identify and connect with the brand. The perception of color often influences the choices a user makes within a site or application, so the goal is to be consistent in the use of the brand color palette.

Brand colors can be used in striking or subtle variations, depending on how and where they are used. Colors that go beyond the brand color palette should be applied only when necessary.

Semantic styling is based on design intent and sensitivity, and can be used freely to get the best results in each scenario. However, make sure to be precise and convey meaning; do not overplay styling based only on visual effect.

Semantic Styling

Neutral: Neutral styling is the default and should be applied whenever there is no strong reason to do otherwise

Positive: Positive styling is reserved for particularly important actions. There should only be one positive button per context

Negative: Negative styling implies a significant negative input; they should only be used where a negative input has serious, lasting effects that the user needs to be warned about. Examples include: aborting a download, canceling a subscription, submitting a negative rating, etc

Subdued: Subdued styling is not the default; they are used to intentionally downplay secondary actions like “Back” or “Cancel”.
